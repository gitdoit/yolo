## 一、输出目录总览

```
├── fire_train/            ← 训练过程的全部输出（最重要）

│   │   ├── best.pt        ← 验证集上表现最好的模型权重（部署用这个）
│   │   └── last.pt        ← 最后一轮的模型权重（可用于恢复训练）

│   ├── results.csv        ← 每轮训练的详细指标数据

│   ├── BoxP_curve.png     ← 精确率曲线

│   ├── train_batch*.jpg   ← 训练批次的可视化（看数据增强效果）
│   ├── val_batch*_labels.jpg  ← 验证集真实标注

│
├── fire_train_val/        ← 评估（验证）的输出
│   ├── BoxF1_curve.png    ← 评估时的 F1 曲线
│   ├── BoxPR_curve.png    ← 评估时的 PR 曲线
│   ├── BoxP_curve.png     ← 评估时的精确率曲线
│   ├── BoxR_curve.png     ← 评估时的召回率曲线

│
└── fire_train_predict/    ← 预测测试的输出
    ├── image0.jpg         ← 测试图片的预测结果（带检测框）
    ├── image1.jpg
    ├── image2.jpg
    ├── image3.jpg
    └── image4.jpg
```

### 三个目录的关系

| 目录 | 来源 | 用途 |
|------|------|------|
| `fire_train/` | `model.train()` | 训练过程产生，包含模型权重和训练指标 |
| `fire_train_val/` | `model.val()` | 用训练好的最佳模型在验证集上做评估 |
| `fire_train_predict/` | `model.predict()` | 用训练好的模型对测试图片做推理，直观看效果 |

## 二、核心评估指标详解

### 2.1 Loss（损失函数）—— 越小越好

| 指标 | 含义 | 通俗解释 |
|------|------|----------|
| `box_loss` | 检测框定位损失 | 模型画的框和真实框差多远（位置+大小） |
| `cls_loss` | 分类损失 | 模型判断"这是火还是烟"判断得对不对 |
| `dfl_loss` | 分布焦点损失 | 检测框边界预测的精细程度 |

**训练时的 loss（train/xxx_loss）应该持续下降**，这说明模型在"学习"。

**验证时的 loss（val/xxx_loss）也应该下降**，如果训练 loss 下降但验证 loss 上升，说明**过拟合**了。

### 2.2 Precision & Recall —— 精确率与召回率

| 指标 | 含义 | 通俗解释 |
|------|------|----------|
| **Precision（精确率）** | 模型说"这是火"，有多少次是对的 | 高精确率 = 很少误报 |
| **Recall（召回率）** | 真实的火，模型找到了多少 | 高召回率 = 很少漏报 |

**Java 工程师理解方式：**
- Precision 高 → 日志中标记为 ERROR 的，确实都是真正的错误
- Recall 高 → 所有真正的错误，都被标记成了 ERROR
- 理想情况两者都高，但实际中往往需要权衡

### 2.3 mAP（Mean Average Precision）—— 最重要的综合指标

| 指标 | 含义 | 说明 |
|------|------|------|
| **mAP50** | IoU≥0.5 时的平均精度 | "框画得大致对就算检测到了" |
| **mAP50-95** | IoU 从 0.5 到 0.95 的平均 | "框要画得很精确才算"（更严格）|

> **IoU（Intersection over Union）**：预测框和真实框的重叠比例。
> IoU=0.5 表示重叠 50% 以上就算"检测到了"。

**mAP 越高越好，1.0 = 完美。一般来说：**
- mAP50 > 0.5：模型有基本的检测能力
- mAP50 > 0.7：效果不错
- mAP50 > 0.9：非常好

### 2.4 F1 Score

$$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$

F1 是精确率和召回率的调和平均，综合考虑了两者。F1 越高越好。

## 三、本次训练结果分析

### 3.1 关键指标走势（来自 results.csv）

| Epoch | box_loss | cls_loss | mAP50 | mAP50-95 | 说明 |
|-------|----------|----------|-------|----------|------|
| 1     | 1.867    | 3.848    | 0.057 | 0.028    | 刚开始，什么都不会 |
| 5     | 1.956    | 3.301    | 0.187 | 0.072    | 开始有感觉了 |
| 10    | 1.864    | 2.902    | 0.446 | 0.155    | 明显进步 |
| 15    | 1.760    | 2.745    | 0.399 | 0.143    | 有波动，正常 |
| 20    | 1.667    | 2.417    | 0.489 | 0.215    | 稳步提升 |
| 25    | 1.672    | 2.429    | 0.560 | 0.233    | 继续提升 |
| 30    | 1.468    | 2.321    | 0.588 | 0.246    | 最终结果 |

### 3.2 结果解读

**好的方面：**
- ✅ train loss（box、cls、dfl）整体呈下降趋势 → 模型确实在学习
- ✅ mAP50 从 0.057 提升到 0.588 → 精度提升了约 10 倍
- ✅ mAP50-95 从 0.028 提升到 0.246 → 框的精确度也在提升
- ✅ 训练完成了 30 轮，没有触发早停 → 模型还在持续学习

**需要注意的：**
- ⚠️ mAP50 = 0.588，还不算很高（0.7+ 才算不错）
- ⚠️ 指标有波动（比如 epoch 15 比 epoch 10 还低），这在小数据集上是正常的
- ⚠️ 数据集比较小（训练集仅 121 张），如果数据更多效果会更好

### 3.3 改进建议

| 方法 | 预期效果 | 难度 |
|------|----------|------|
| 增加训练轮数到 50~100 | loss 可能继续下降 | ⭐ 简单 |
| 使用更大模型 (yolov8s/yolov8m) | 模型容量更大，可能学得更好 | ⭐ 简单 |
| 使用 yolo26n 替代 yolov8n | 更新架构，可能更高效 | ⭐ 简单 |
| 增加数据量（Roboflow 扩充） | 最有效的提升方式 | ⭐⭐ 中等 |
| 调整数据增强策略 | 让小数据集发挥更大作用 | ⭐⭐ 中等 |
| 调整超参数（lr、optimizer等） | 精细优化 | ⭐⭐⭐ 较难 |

## 四、每个输出文件详解

### 4.1 results.png — 训练曲线总图

**最常看的一张图**，包含多个子图：
- 左侧：train loss 和 val loss 随 epoch 的变化
- 右侧：precision、recall、mAP50、mAP50-95 随 epoch 的变化

**怎么看：**
- loss 曲线应该向下（在学习）
- mAP 曲线应该向上（效果在变好）
- 如果 train loss 下降但 val loss 上升 → 过拟合

### 4.2 confusion_matrix.png — 混淆矩阵

展示模型把哪些类别搞混了。

```
              预测
              exist  fire  smoke  background
真  exist       ✓     ✗      ✗       ✗
实  fire        ✗     ✓      ✗       ✗
    smoke       ✗     ✗      ✓       ✗
```

- 对角线上的数字越大越好（预测对了）
- 非对角线的数字越小越好（预测错了）
- `confusion_matrix_normalized.png` 是百分比版本，更容易看

### 4.3 BoxPR_curve.png — PR 曲线

横轴是 Recall，纵轴是 Precision。

- 曲线越靠近右上角越好（又准又全）
- 曲线下方的面积就是 AP（Average Precision）
- 每个类别一条线，可以看出哪个类别检测效果最好/最差

### 4.4 BoxF1_curve.png — F1 曲线

横轴是置信度阈值，纵轴是 F1 分数。

- 曲线的峰值对应最佳置信度阈值
- 部署时可以参考这个峰值来设置 `conf` 参数

### 4.5 labels.jpg — 数据集标签分布

展示数据集中各类别的数量分布和标注框的位置/大小分布。

- 如果某个类别样本极少，模型可能学不好该类别
- 如果标注框都集中在图片中央，模型可能对边缘区域的目标检测不好

### 4.6 train_batch*.jpg — 训练批次可视化

展示训练时输入模型的图片（经过数据增强后的样子）。

- 可以确认数据增强是否合理
- 可以确认标注是否正确

### 4.7 val_batch*_labels.jpg vs val_batch*_pred.jpg

| 文件 | 内容 |
|------|------|
| `val_batch*_labels.jpg` | 验证集图片 + **真实标注**（Ground Truth） |
| `val_batch*_pred.jpg` | 验证集图片 + **模型预测**（Prediction） |

**对比这两组图片是最直观的评估方式** —— 看模型画的框和真实框差多少。

### 4.8 weights/ — 模型权重

| 文件 | 用途 |
|------|------|
| `best.pt` | 训练过程中验证集指标最好的那一轮的权重 → **部署用这个** |
| `last.pt` | 最后一轮的权重 → 可用于恢复训练（`resume=True`） |

### 4.9 args.yaml — 训练参数记录

完整记录了本次训练的所有参数，方便以后复现实验。

### 4.10 results.csv — 详细指标数据

每轮训练的所有指标，用逗号分隔，可以用 Excel 或 Python 打开做进一步分析。

## 五、fire_train vs fire_train_val vs fire_train_predict

| | fire_train | fire_train_val | fire_train_predict |
|--|-----------|---------------|-------------------|
| **产生方式** | `model.train()` | `model.val()` | `model.predict()` |
| **目的** | 训练模型 | 评估模型精度 | 实际推理看效果 |
| **包含模型权重** | ✅ 有 | ❌ 无 | ❌ 无 |
| **包含训练指标** | ✅ 有 (results.csv) | ❌ 无 | ❌ 无 |
| **包含评估指标** | ✅ 有（训练时自动验证） | ✅ 有（单独评估） | ❌ 无 |
| **包含预测图片** | ❌ 无 | ❌ 无 | ✅ 有（带检测框） |
| **使用场景** | 训练过程记录 | 精确评估最终模型 | 直观展示检测效果 |

**总结：**
- `fire_train/` 是"训练日志"，记录了整个训练过程
- `fire_train_val/` 是"考试成绩单"，用最好的模型做一次正式评估
- `fire_train_predict/` 是"实战演练"，拿几张图片做预测看实际效果

## 六、快速评估检查清单

训练完成后，按以下顺序检查：

1. **打开 `results.png`** → loss 在下降吗？mAP 在上升吗？
2. **打开 `confusion_matrix_normalized.png`** → 哪些类别容易搞混？
3. **对比 `val_batch0_labels.jpg` 和 `val_batch0_pred.jpg`** → 框画得准吗？
4. **查看 `fire_train_predict/` 下的图片** → 实际检测效果怎么样？
5. **看 `results.csv` 最后一行的 mAP50** → 是否达到期望的精度？

如果效果不满意，参考第三节的改进建议进行调优。
